# _daily 작업 규칙

## 블로그용 학습 내용 정리

사용자가 "블로그에 쓸 내용 정리해줘", "오늘 배운 내용 정리해줘" 같은 요청을 하면, 그 대화에서 다룬 학습 내용을 정리해 블로그에 바로 옮겨 쓸 수 있는 마크다운 글로 작성한다.

### 파일 위치 및 이름
- `hancom/_daily/<오늘날짜 YYMMDD>-blog.md` (기존 `hancom/_daily/<YYMMDD>.md` 할 일 메모와 같은 날짜 표기 규칙 사용)
- 하루에 여러 번 요청할 수 있으며, 하루치 블로그 글에는 여러 주제가 함께 담길 수 있다.
  - 같은 날 파일이 이미 있으면 덮어쓰거나 새 파일을 만들지 말고, 그날 배운 새 주제를 이어서(섹션 추가) 작성한다.
  - 파일이 없으면 새로 생성한다.

### 작성 방식
- 실제 대화에서 다룬 내용만 근거로 작성한다 (없는 내용을 지어내지 않는다).
- 마지막에 짧은 요약/정리로 마무리한다.
- 한국어로 작성한다.

### 제목 레벨

- 파일 전체의 H1은 `# 오늘 학습한 내용` 하나만 쓴다. 하루 파일 전체가 그날 올릴 블로그 글 한 편이다.
- 그날 다룬 주제는 H2(`##`)로 구분한다.
- 각 주제 안의 소제목(원래 실습 방식/시도한 방식/개념 설명 등)은 H3(`###`) 이하로 작성한다.
- 새 주제를 이어 붙일 때도 이 레벨 규칙을 유지한다 (H1을 새로 추가하지 않는다).
- H3(또는 그 하위 제목) 안에 서로 구분되는 항목이 2개 이상 있고 각 항목마다 별도로 설명할 내용이 있다면, 그 항목들을 한 문단/글머리 기호 목록으로 뭉치지 말고 각각 H4(`####`) 이하 소제목으로 나눠 작성한다.
  - 특정 섹션(예: "개선" 섹션)에만 적용되는 규칙이 아니라, 원래 코드 설명/개념 설명/결론 등 어떤 섹션이든 항목이 여러 개로 나뉠 수 있으면 동일하게 적용한다.
  - 단순히 한 줄짜리 항목을 나열하는 짧은 목록(예: 요약 정리, 특징 나열)까지 전부 쪼개라는 뜻은 아니며, 항목마다 별도의 설명·코드가 붙어 실질적으로 하나의 소주제로 볼 수 있는 경우에 적용한다.
