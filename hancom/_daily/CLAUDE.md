# _daily 작업 규칙

## 블로그용 학습 내용 정리

사용자가 "블로그에 쓸 내용 정리해줘", "오늘 배운 내용 정리해줘" 같은 요청을 하면, 그 대화에서 다룬 학습 내용을 정리해 블로그에 바로 옮겨 쓸 수 있는 마크다운 글로 작성한다.

### 파일 위치 및 이름
- `hancom/_daily/<오늘날짜 YYMMDD>-blog.md` (기존 `hancom/_daily/<YYMMDD>.md` 할 일 메모와 같은 날짜 표기 규칙 사용)
- 하루에 여러 번 요청할 수 있으며, 하루치 블로그 글에는 여러 주제가 함께 담길 수 있다.
  - 같은 날 파일이 이미 있으면 덮어쓰거나 새 파일을 만들지 말고, 그날 배운 새 주제를 이어서(섹션 추가) 작성한다.
  - 파일이 없으면 새로 생성한다.

### 작성 방식
- 실제 대화에서 다룬 내용만 근거로 작성한다 (없는 내용을 지어내지 않는다).
- 주제별로 소제목을 달고, 필요하면 코드 스니펫을 포함한다.
- 마지막에 짧은 요약/정리로 마무리한다.
- 한국어로 작성한다.
