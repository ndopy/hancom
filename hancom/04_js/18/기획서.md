# 카드 매칭 게임 (Memory Card Game) 기획서

## 1. 프로젝트 개요

- **주제**: 이모지를 활용한 카드 짝맞추기(메모리 매칭) 게임
- **목적**: JS `class` 문법을 실제로 활용해보는 학습 목적의 미니 프로젝트
- **개발 기간**: 2026-07-07 ~ 2026-07-08 16:00
- **배포**: Netlify

## 2. 주제 선정 배경

그동안 JS 개발을 하면서 대부분의 로직을 함수 위주로 해결해와서, `class` 문법을 실질적으로 어떻게 활용하는지 체감하기 어려웠다. 카드 매칭 게임은 카드 한 장 한 장이 각자 독립적인 상태(뒤집힘 여부, 매칭 여부)와 행동(뒤집기 등)을 가지므로, `class`로 표현했을 때 장점이 뚜렷하게 드러나는 주제라 선정했다.

## 3. 주요 기능

### MVP (필수)

- 게임 시작 버튼 클릭 시 게임 시작 (그 전엔 카드 비공개/비활성)
- 게임 시작 시점부터 1초 단위로 증가하는 타이머 표시
- 카드 16장(8쌍) 랜덤 배치
- 카드 클릭 시 뒤집기 애니메이션
- 두 장 클릭 시 일치 여부 판별 (일치 시 유지, 불일치 시 다시 뒤집기)
- 모든 카드 매칭 완료 시 승리 메시지 표시 및 타이머 정지
- 시도 횟수 카운트

### MVP 이후 (여유 있으면)

- 재시작 버튼

## 4. 사용 기술

- HTML5, CSS3 (Flexbox/Grid, transition/animation)
- Vanilla JavaScript (ES6 Class)
- 카드 심볼은 이모지 사용 (별도 이미지 에셋 없음)
- Netlify 배포

## 5. 데이터 구조 설계 (초안)

- **Card 클래스**: 카드 한 장의 상태(뒤집힘 여부, 매칭 여부, 심볼 등) 관리
- **Game(또는 Board) 클래스**: 전체 카드 목록, 시도 횟수, 게임 진행 상태 관리

> 정확한 필드/메서드 구성은 구현 단계에서 직접 설계.

## 8. AI(Claude) 활용 방침

이번 프로젝트의 핵심 목적은 `class` 문법을 직접 체득하는 것이므로, 학습 목표와 직결되는 부분에서는 Claude 활용을 최소화하는 방향으로 진행한다.

- **Card 클래스 설계 및 게임 로직(뒤집기, 매칭 판별, 타이머, 시도 횟수 등)**: 코드는 직접 작성하고, Claude에게는 막힌 부분에 대한 개념 설명이나 힌트, 작성한 코드의 버그 리뷰 정도만 요청한다.
- **CSS 디자인, 기획서 문서화 등 class 문법 학습과 직접 관련이 적은 영역**: 상대적으로 Claude의 도움을 더 적극적으로 활용한다.
