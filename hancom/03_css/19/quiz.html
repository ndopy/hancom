3. 퀴즈

정답을 맞히는 것보다, 실제로 헷갈리기 쉬운 개념을 짚어보는 데 목적을 두고 문제를 만들었습니다. 객관식은 선택지를 고르면 바로 정답이 나타나고, 주관식은 먼저 답을 생각해본 뒤 "정답 및 해설 보기"를 눌러 확인해보세요.

Q1. [객관식] 다음 중 특이성(specificity)이 가장 높아서 최종적으로 적용되는 CSS 규칙은 무엇일까요?

정답: D

우선순위는 인라인 스타일 > id 선택자 > class/속성/가상클래스 선택자 > 태그 선택자 순입니다. id가 가장 강력하다고 착각하기 쉬운데, 요소에 직접 붙인 인라인 스타일이 그보다도 우선합니다.

Q2. [주관식] 어떤 요소에 width: 200px; padding: 20px; border: 5px solid;가 적용되어 있습니다. box-sizing: content-box(기본값)일 때와 box-sizing: border-box일 때, 이 요소가 화면에서 실제로 차지하는 전체 너비는 각각 몇 px일까요?

정답: content-box → 250px / border-box → 200px

content-box는 width가 content 영역만의 너비라서, 여기에 padding(20px × 2)과 border(5px × 2)가 더해져 200 + 40 + 10 = 250px가 됩니다. border-box는 padding과 border가 이미 width 안에 포함되도록 계산하므로, 지정한 200px가 그대로 전체 너비가 됩니다.

Q3. [객관식] 스크롤하는 동안에는 원래 자리에 있다가, 특정 지점을 지나면 화면에 고정되어 따라오는 position 값은 무엇일까요?

정답: D

fixed는 스크롤 위치와 상관없이 처음부터 항상 화면(뷰포트)에 고정됩니다. 반면 sticky는 지정한 임계점(예: top: 0)에 도달하기 전까지는 relative처럼 원래 자리를 지키다가, 그 지점을 넘어가는 순간부터 고정되는 요소입니다. 이 둘을 같은 것으로 헷갈리기 쉽습니다.

Q4. [주관식] 아래처럼 <p> 태그 안에 <ul>을 넣으면 브라우저는 이 코드를 실제로 어떻게 처리할까요? 그리고 왜 그런 문제가 생기는지 설명해보세요. 

<p>
  <ul>
    <li>...</li>
  </ul>
</p>

정답: <p>는 인라인 레벨 콘텐츠만 담을 수 있어서, <ul>(블록 레벨 요소)을 만나는 순간 브라우저가 <p>를 자동으로 닫아버립니다. 그 뒤에 남은 </p>는 짝이 없는 태그가 되어버립니다.

실제로 이 프로젝트의 "웹사이트 분석" 페이지를 작성하다가 정확히 이 문제로 Prettier가 "Unexpected closing tag p" 에러를 냈던 적이 있습니다. <ul>을 <p> 밖으로 꺼내서 형제 요소로 분리하니 해결됐습니다.

Q5. [객관식] 모바일 퍼스트 방식으로 CSS를 작성할 때, 화면이 커질수록 스타일을 "추가"해나가기 위해 주로 사용하는 미디어쿼리 조건은 무엇일까요?

정답: B

모바일 퍼스트는 기본 스타일을 좁은 화면 기준으로 작성하고, min-width 미디어쿼리로 화면이 일정 너비 이상이 될 때 스타일을 덧붙여 나가는 방식입니다. 반대로 데스크톱을 기본으로 하고 좁아질 때 max-width로 스타일을 덮어쓰는 방식은 데스크톱 퍼스트라고 부릅니다.
