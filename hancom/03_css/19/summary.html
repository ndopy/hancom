2. HTML/CSS 정리

그동안 배운 내용을 전부 나열하기보다, 실무에서도 자주 쓰이고 이번 프로젝트에도 직접 적용해본 "특히 중요한 개념" 위주로 정리했습니다.

HTML 핵심

시맨틱 마크업

정의: 콘텐츠의 의미(역할)를 드러내는 태그(<header>, <nav>, <main>, <section>, <footer> 등)를 목적에 맞게 사용하는 것입니다.

왜 중요한가: 스크린리더 같은 보조기술이 페이지 구조를 이해할 수 있게 해주고(접근성), 검색엔진이 콘텐츠의 우선순위를 파악하는 데도 도움을 줍니다(SEO). 코드만 봐도 어떤 역할의 영역인지 알 수 있어서 유지보수도 쉬워집니다. 

<body>
  <header>...</header>
  <nav>...</nav>
  <main>
    <section>...</section>
  </main>
  <footer>...</footer>
</body>

다만 "웹사이트 분석" 페이지에서 확인했듯, Apple은 <header> 없이 <nav>만으로 상단 영역을 처리하고 있었습니다. 시맨틱 태그를 항상 공식처럼 다 쓰는 게 아니라, 상황에 맞게 유연하게 적용한다는 것도 함께 배웠습니다.

폼 요소

정의: 사용자로부터 입력을 받는 <input>, <textarea>, <select>, <button> 등을 <form> 태그로 묶은 것입니다.

왜 중요한가: 로그인, 검색, 회원가입처럼 사용자와 상호작용하는 대부분의 기능이 폼을 기반으로 합니다. <label>과 <input>을 for/id로 연결하면 접근성도 함께 챙길 수 있습니다. 

<form>
  <label for="email">이메일</label>
  <input type="email" id="email" name="email" required />
  <button type="submit">제출</button>
</form>

CSS 핵심

1) 선택자 우선순위(특이성)

정의: 여러 CSS 규칙이 같은 요소에 적용될 때, 어떤 규칙이 최종적으로 이길지를 정하는 계산 방식입니다.

왜 중요한가: "분명히 스타일을 줬는데 왜 안 먹히지?"라는 문제의 대부분이 특이성 때문에 발생합니다. 인라인 스타일 > id > class/속성/가상클래스 > 태그 순으로 우선순위가 높다는 걸 알아야 원인을 빠르게 찾을 수 있습니다. 

p { color: blue; }          /* 특이성: 0-0-1 */
.text { color: green; }     /* 특이성: 0-1-0 */
#title { color: red; }      /* 특이성: 1-0-0 */

세 규칙이 모두 같은 요소에 적용돼도, id 선택자를 쓴 #title이 항상 이깁니다.

2) 박스모델

정의: 모든 HTML 요소는 content, padding, border, margin으로 이루어진 사각형 박스로 렌더링됩니다.

왜 중요한가: width에 padding과 border가 포함되는지 아닌지(box-sizing)를 모르면 레이아웃 계산이 계속 어긋납니다. 

* { box-sizing: border-box; }

이 한 줄만 넣어도 width에 padding/border가 포함되어 계산이 훨씬 예측 가능해집니다. 이번 프로젝트 CSS 맨 위에도 이 규칙을 가장 먼저 넣었습니다.

3) Flexbox vs Grid

정의: 둘 다 레이아웃을 잡는 CSS지만, Flexbox는 한 줄/한 열 같은 1차원 정렬에, Grid는 행과 열을 함께 다루는 2차원 배치에 특화되어 있습니다.

왜 중요한가: 언제 뭘 써야 할지 헷갈리기 쉬운데, "메뉴를 한 줄로 정렬"하는 건 Flexbox가, "카드 여러 개를 격자로 배치"하는 건 Grid가 훨씬 간단합니다. 

.site-nav { display: flex; gap: 16px; }                              /* 1차원 정렬 */
.card-grid { display: grid; grid-template-columns: repeat(3, 1fr); } /* 2차원 배치 */

실제로 이번 프로젝트에서도 상단 nav 메뉴는 Flexbox로, 홈 화면의 카드 목록은 Grid로 구현했습니다.

4) 반응형(미디어쿼리 + 모바일 퍼스트)

정의: @media는 화면 너비에 따라 다른 스타일을 적용하는 규칙입니다. 모바일 퍼스트는 기본 스타일을 모바일 기준으로 작성하고, min-width 미디어쿼리로 화면이 커질 때 스타일을 "추가"해나가는 방식입니다.

왜 중요한가: 대부분의 트래픽이 모바일에서 발생하고, 좁은 화면부터 설계하면 불필요한 스타일 덮어쓰기가 줄어듭니다. "웹사이트 분석" 페이지에서 확인했듯 Apple도 734px, 1068px 같은 실제 브레이크포인트를 쓰고 있었습니다. 

/* 기본값 = 모바일 */
.card-grid { grid-template-columns: 1fr; }

@media (min-width: 600px) {
  .card-grid { grid-template-columns: repeat(2, 1fr); }
}

5) CSS 변수

정의: --변수명: 값;으로 선언하고 var(--변수명)으로 재사용하는 커스텀 속성입니다.

왜 중요한가: 색상이나 간격 같은 값을 한 곳에서 관리할 수 있어 유지보수가 쉬워지고, 클래스에 따라 같은 변수가 다른 값을 갖게 해서 다크모드 같은 테마 전환도 구현할 수 있습니다. 실제로 "웹사이트 분석" 페이지에서 Apple도 --sk-body-text-color 같은 변수로 라이트/다크 테마를 전환하는 것을 확인했습니다. 

:root {
  --color-accent: #0066cc;
}

.site-nav a[aria-current="page"] {
  color: var(--color-accent);
}

6) 애니메이션(transition / @keyframes)

정의: transition은 hover 같은 상태 변화를 부드럽게 이어주고, @keyframes는 여러 단계를 거치는 자동 애니메이션을 정의합니다.

왜 중요한가: 사용자에게 "무언가 바뀌었다"는 피드백을 자연스럽게 전달해서 인터랙션의 완성도를 높여줍니다. 

.card {
  transition: border-color 0.2s, transform 0.2s;
}

.card:hover {
  transform: translateY(-2px);
}
