# 저장소 공통 작업 규칙

이 파일은 저장소 전체에 적용되는 공통 규칙이다. 특정 폴더에만 적용되는 규칙은 해당 폴더의 `CLAUDE.md`(예: `hancom/04_js/CLAUDE.md`, `hancom/03_css/CLAUDE.md`, `hancom/_daily/CLAUDE.md`)를 따른다.

## 응답 언어

- 이 저장소에서 작업할 때는 항상 한국어로 설명한다.
- 코드, 식별자, 커밋 메시지 제목 등에 등장하는 영어 키워드(`const`, `addEventListener` 등)는 그대로 두되, 설명하는 문장 자체는 한국어로 작성한다.

## Git 커밋 컨벤션

- 커밋 메시지는 항상 **제목 + 빈 줄 + 본문(1~3줄 설명)**으로 멀티라인 작성한다. 제목 한 줄만 있는 커밋은 만들지 않는다.
  - 본문에는 무엇을 구현/변경했는지와 주목할 만한 설계 선택을 한국어로 짧게 적는다.
- 여러 폴더나 서로 다른 주제의 작업을 한 커밋에 묶지 않는다. 폴더·주제 단위로 각각 별도 커밋을 만든다.
  - 예: `hancom/04_js/08`과 `hancom/04_js/09`는 각각 별도 커밋, 규칙 문서(`CLAUDE.md`)와 실제 콘텐츠(`_daily/*.md`)도 서로 다른 커밋으로 분리한다.
- 사용자가 명시적으로 요청했을 때만 커밋하고, push는 별도로 명시적인 요청이 있을 때만 진행한다.
